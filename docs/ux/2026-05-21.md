## Auth (login) (`/auth`)

# 🔴 Auth Page Review – Top 5 Issues

## 1. 🔴 **Página de Auth inexistente / Error 404 encubierto**

**Problema:**  
El HTML es el shell vacío de React. La ruta `/auth` probablemente no existe o no renderiza nada. Un Head of Ops que recibe un link de invitación y cae aquí ve pantalla blanca = bounce inmediato.

**Fix:**
```tsx
// App.tsx o router config
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom';
import AuthPage from './pages/AuthPage';

<Routes>
  <Route path="/auth" element={<AuthPage />} />
  <Route path="*" element={<Navigate to="/auth" replace />} />
</Routes>
```

**Validación crítica:**  
- Crear `/auth` con Google OAuth (Clerk/Auth0/Supabase)
- Si usuario ya autenticado, redirect a `/search` automático
- Si no existe cuenta, permitir "Request Access" → Typeform/Cal.com

**Problema:**  
Si OAuth falla (permisos de Google Workspace bloqueados, email no whitelisted, etc.), el usuario queda en el limbo. En B2B esto mata conversión porque el Head de Ops piensa "no funciona" y ghostea.

**Fix:**
```tsx

{authError && (
  <div 
    role="alert"
    className="mb-6 rounded-lg border-l-4 border-red-500 bg-red-50 p-4"

    <p className="text-sm font-medium text-red-800">

      ¿Necesitas ayuda? Agenda 15 min con nosotros →

  </div>

```

**Mensajes de error en español natural:**
- ❌ "Authentication failed"  
- ✅ "No pudimos verificar tu cuenta. ¿Usaste el email corporativo?"
